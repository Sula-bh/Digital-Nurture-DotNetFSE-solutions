1. Explain linear search and binary search algorithms.

Answer:

The Search Algorithms are:

Linear Search: Sequentially checks each element until the target is found or the list ends. It works on both sorted and unsorted data.
Binary Search: Repeatedly divides the sorted dataset into two halves and searches only the relevant half until the target is found.

2. Compare the time complexity of linear and binary search.

Answer:

The Time Complexity Comparison is:

Case			Linear Search	Binary Search
Best Case		O(1)			O(1)
Average Case	O(n)			O(log n)
Worst Case		O(n)			O(log n)

Where:

n = Number of books in the library.

3. Discuss when to use each algorithm based on the data set size and order.

Answer:

The Appropriate Use Cases are:

Linear Search:
Suitable for small datasets.
Works on unsorted data.
Easy to implement.

Binary Search:
Suitable for large datasets.
Requires sorted data.
Provides significantly faster search performance.

For large library collections with sorted records, Binary Search is preferred due to its O(log n) time complexity. For small or unsorted collections, Linear Search is more practical.
